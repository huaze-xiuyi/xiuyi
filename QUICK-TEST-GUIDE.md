# 深色模式优化 - 快速测试指南

**时间：** 5-10 分钟
**目的：** 验证深色模式色彩优化是否正确应用

---

## 🚀 快速开始（2 分钟）

### Step 1: 刷新页面并切换深色模式

1. 打开任一页面（如 `index.html`）
2. 刷新浏览器（Ctrl+F5 清除缓存）
3. 点击右上角 🌙 图标切换为深色模式

### Step 2: 视觉检查（2 分钟）

观察以下是否正确：

- [ ] 背景色深黑蓝 (#0f172a) - **看起来应该很深但不是纯黑**
- [ ] 标题文本纯白 (#ffffff) - **应该非常清晰、清楚**
- [ ] 正文文本浅灰 (#cbd5e1) - **应该易于阅读，不闪烁**
- [ ] 按钮浅青蓝 (#4dd0fc) - **应该比之前更亮、更清晰**
- [ ] 按钮悬停 (#7dd3fc) - **应该更亮，提供视觉反馈**
- [ ] 卡片背景 (#1a1f36) - **应该略亮于主背景，有层次感**

### Step 3: 对比度验证（2 分钟）

打开检查工具：
**文件：** `darkmode-contrast-checker.html`

1. 在浏览器中打开该文件
2. 查看"对比度合规性检查"部分
3. 验证所有元素都有 ✓ 绿色对比度标识

## 🔍 详细测试（5-10 分钟）

### 测试 1: 文本清晰度

**操作：**
1. 打开 `shanghai-happiness-dashboard.html`
2. 切换到深色模式
3. 阅读以下位置的文本：
   - 页面标题
   - 图表标签
   - 表格内容
   - 卡片描述

**验证标准：**
- [ ] 标题（应该是白色 #ffffff）清晰锐利
- [ ] 副标题（应该是 #e2e8f0）易于阅读
- [ ] 正文（应该是 #cbd5e1）舒适流畅

---

### 测试 2: 按钮交互

**操作：**
1. 在深色模式下查看所有按钮
2. 悬停在按钮上观察变化

**验证标准：**

#### 主按钮（CTA）
- [ ] 正常态：浅青蓝色 (#4dd0fc)
- [ ] 悬停态：更亮的浅青 (#7dd3fc)
- [ ] 点击态：轻微缩小（scale 0.98）
- [ ] 点击态阴影：变浅

#### 次按钮
- [ ] 正常态：浅青色带半透明背景
- [ ] 悬停态：背景加深，边框加粗
- [ ] 文本颜色：#4dd0fc

#### 危险按钮
- [ ] 正常态：红色渐变
- [ ] 悬停态：阴影加强
- [ ] 文本：纯白

---

### 测试 3: 玻璃态效果

**操作：**
1. 观察卡片、弹窗等玻璃态元素
2. 查看边框是否清晰

**验证标准：**
- [ ] 背景：略透明但清晰（rgba(26, 31, 54, 0.92)）

- [ ] 内阴影：细微但增加质感
- [ ] 悬停时：背景稍亮，边框更清晰

## 🆘 问题排查

### 问题 1: 颜色没有改变

**症状：** 深色模式仍然显示旧的颜色

**排查步骤：**
1. 按 `Ctrl+Shift+Delete` 清除浏览器缓存
2. 刷新页面（Ctrl+F5）
3. 检查 `styles.css` 是否已保存
4. 在浏览器开发者工具中搜索 `--color-primary`，确认值为 `#4dd0fc`

### 问题 2: 文本不清晰

**症状：** 文本显得模糊或闪烁

**排查步骤：**
1. 确认文本颜色变量已应用：检查 `--text-primary`, `--text-secondary` 等
2. 检查是否有其他 CSS 规则覆盖了文本颜色（搜索 `color: #`）
3. 禁用浏览器扩展（可能影响样式）
4. 尝试不同的浏览器

### 问题 3: 按钮悬停效果不明显

**症状：** 悬停时按钮变化不明显

**排查步骤：**
1. 确认悬停 CSS 已添加到 `styles.css`（第 194-246 行）
2. 检查是否有 CSS 优先级问题（使用 `!important` 强制）
3. 验证 CSS 类名是否正确（`.btn-primary`, `.btn-secondary` 等）

### 问题 4: 玻璃态边框看不清

**症状：** 玻璃态卡片的边框不可见

**排查步骤：**
1. 检查边框颜色：应该是 `rgba(148, 163, 184, 0.5)`
2. 尝试增加不透明度：改为 `rgba(148, 163, 184, 0.7)`
3. 检查浏览器是否支持 `backdrop-filter: blur()`（IE 11 不支持）
